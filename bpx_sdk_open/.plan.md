# BPX SDK 图形化编程系统 — 实现方案

## 一、总体架构

```
┌─────────────────────────────────────────────────────┐
│                   浏览器 (前端)                        │
│  ┌──────────────┐  ┌──────────────┐  ┌────────────┐ │
│  │  Blockly 积木 │  │  3D 可视化    │  │  状态仪表盘 │ │
│  │  编辑器       │  │  (Three.js)  │  │  (Canvas)  │ │
│  └──────┬───────┘  └──────┬───────┘  └─────┬──────┘ │
│         │                 │                 │        │
│         └──────── WebSocket ────────────────┘        │
└─────────────────────────┬───────────────────────────┘
                          │ ws://localhost:8080
┌─────────────────────────┴───────────────────────────┐
│               C++ 单进程后端                           │
│  ┌──────────────┐  ┌──────────────┐  ┌────────────┐ │
│  │ HTTP 静态文件  │  │ WebSocket    │  │ 程序执行引擎 │ │
│  │ 服务 (前端)    │  │ 双向通信     │  │ (AST解释器) │ │
│  └──────────────┘  └──────────────┘  └─────┬──────┘ │
│                                            │        │
│  ┌─────────────────────────────────────────┴──────┐ │
│  │              BPX SDK 封装层                      │ │
│  │  MotionLevelControl / JointLevelControl         │ │
│  └────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
                          │ UDP/TCP
                    ┌─────┴─────┐
                    │  BPX 机器狗 │
                    └───────────┘
```

**单进程部署**：C++ 后端同时提供 HTTP 静态文件服务（前端页面）和 WebSocket 实时通信，无需 nginx/Node.js 等额外依赖。

## 二、网络接口配置

用户可通过**有线网口、无线 WiFi、USB 网络共享**等多种方式连接机器狗，系统需要支持灵活的网络接口选择与配置。

### 2.1 后端网络配置

```cpp
// 启动时可配置参数
struct NetworkConfig {
    std::string robot_ip = "10.21.20.1";           // 机器狗 IP（有线默认 10.21.20.1，无线默认 192.168.0.1）
    std::string local_interface = "auto";           // 本地网卡名，如 "eth0"、"wlan0"，"auto" 为自动检测
    uint16_t state_udp_port = 9873;                 // 本地 UDP 接收端口
    uint16_t joint_state_udp_port = 7895;           // 高频关节数据 UDP 端口
    uint16_t tcp_local_port = 0;                    // TCP 本地端口，0 = 自动
    uint16_t state_upload_rate = 100;               // 状态上传频率 (Hz)
    uint16_t motion_command_rate = 50;              // 运动指令发送频率 (Hz)
    uint16_t web_server_port = 8080;                // Web 服务端口
    std::string web_server_bind = "0.0.0.0";        // Web 服务绑定地址
};
```

**启动方式**：
```bash
# 使用默认配置（自动检测网卡）
./bpx_visual_programming

# 指定网卡和参数
./bpx_visual_programming --interface eth0 --robot-ip 10.21.20.1 --web-port 8080

# 无线连接时指定
./bpx_visual_programming --interface wlan0 --robot-ip 192.168.0.1

# 或通过配置文件
./bpx_visual_programming --config config.json
```

**关键特性**：
- 网卡列表**实时动态刷新**，显示当前所有可用网络接口
- 每个接口显示：名称、IP 地址、类型图标（🔌有线 / 📶无线 / 🔗USB）
- 自动标记与机器人 IP 同网段的接口（带 ✓ 提示）
- 支持**手动输入**自定义本地 IP（用于特殊网络环境）
- 连接状态实时显示，切换网卡后可一键重连

### 2.3 网络接口自动检测

后端启动时自动枚举本机所有网络接口，返回完整列表供前端选择：

```cpp
struct NetworkInterface {
    std::string name;           // "eth0" / "wlan0" / "usb0"
    std::string ip;             // "10.21.20.100"
    std::string mac;            // "aa:bb:cc:dd:ee:ff"
    std::string type;           // "ethernet" / "wireless" / "usb" / "other"
    bool is_up;                 // 是否已启用
    bool same_subnet_as_robot;  // 是否与机器人同网段
};
```

## 三、技术选型

| 组件 | 技术方案 | 说明 |
|------|---------|------|
| 前端积木编辑器 | **Google Blockly** | 最成熟的积木编程库，支持自定义 block、toolbox、代码生成 |
| 前端 3D 可视化 | **Three.js** | 轻量级 WebGL 库，用于机器狗关节 3D 模型渲染 |
| 前端仪表盘 | **HTML5 Canvas** | 电池、IMU 姿态、关节角度仪表盘 |
| 嵌入式 HTTP 服务器 | **cpp-httplib** | 单头文件库，零依赖，支持 HTTP + 静态文件服务 |
| WebSocket 服务器 | **uWebSockets** | 高性能 C++ WebSocket 库（或降级方案：简易手写 WebSocket） |
| JSON 处理 | **nlohmann/json** | 单头文件 JSON 库，C++17 友好 |
| 构建系统 | **CMake** | 扩展现有 CMakeLists.txt |

## 七、核心模块设计

### 7.1 RobotBridge（SDK 封装层）

```
class RobotBridge {
    MotionLevelControl motion_ctrl_;
    JointLevelControl joint_ctrl_;
    NetworkConfig net_config_;
    bool motion_connected_ = false;
    bool joint_connected_ = false;

    // 网络配置
    void setNetworkConfig(const NetworkConfig& config);
    std::vector<NetworkInterface> getAvailableInterfaces() const;
    bool setInterface(const std::string& interface_name);

    // 连接管理（根据 net_config_ 中的 ip 和 interface 自动绑定）
    bool connect();
    bool connect(const std::string& ip, const std::string& interface = "auto");
    void disconnect();
    bool isConnected() const;

    // 运动控制
    bool standUp();
    bool sitDown();
    bool setDamping();
    bool setVelocity(float x, float y, float yaw);

    // 关节控制
    bool setJointPosition(const std::array<float, 12>& pos);
    bool setJointGains(float kp, float kd);
    bool smoothMoveTo(const std::array<float, 12>& target, float duration_sec);

    // 状态读取
    json getState() const;  // 返回完整状态 JSON
};
```

### 7.2 CommandExecutor（程序执行引擎）

```
class CommandExecutor {
    // 接收 Blockly 生成的 JSON 指令数组，顺序执行
    void execute(const json& program, RobotBridge& bridge);

    // 支持的指令类型：
    // - connect, disconnect
    // - standUp, sitDown, damping, upright
    // - setVelocity, wait
    // - setJointPosition, setJointGains, smoothMove, zeroJoints
    // - loop (repeat N times / until condition)
    // - if/else (基于传感器条件)
    // - emergencyStop

    // 执行控制
    void stop();   // 紧急停止
    bool isRunning() const;
};
```

### 7.3 StatePublisher（状态推送）

```
class StatePublisher {
    // 以固定频率（10Hz）将机器人状态通过 WebSocket 推送给所有客户端
    void start(std::function<void(const json&)> broadcast);
    void stop();

    // 状态采样：SDK 高频数据 → 10Hz 推送
    // 包含：关节数据、IMU、电池、运动状态、温度等
};
```

## 十、安全设计

| 安全措施 | 说明 |
|---------|------|
| 关节角度限位 | 执行前检查目标角度是否在安全范围内 |
| 速度限制 | 限制最大速度值，防止失控 |
| 紧急停止按钮 | 页面醒目的红色停止按钮，立即发送阻尼模式 |
| 低电量保护 | 电量 < 15% 自动停止执行并警告 |
| 连接超时 | WebSocket 断开自动进入阻尼模式 |
| 执行日志 | 所有指令执行均有日志记录 |
